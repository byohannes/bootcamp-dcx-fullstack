# Sprint 0 — Setup & Planning

## Goals

Get the development environment running, understand the project structure, and plan the implementation.

## Tasks

- [ ] Clone the repository
- [ ] Install dependencies (`nvm use && npm install`)
- [ ] Run the bike booking app (`npm run dev:bike`)
- [ ] Verify client loads at http://localhost:5174
- [ ] Verify API health check at http://localhost:5001/api/health
- [ ] Understand the folder structure
- [ ] Review PRD requirements
- [ ] Create feature branch for development

## PRD Overview

**Product**: Responsive web-based bike booking application
**Users**: Individuals renting bikes for short periods (hours or a day)
**Problem Solved**: Manual/unclear bike booking → easy online availability check and reservation

### Core Features (Sprints 1-2)

1. View Available Bikes
2. Select Date/Time for Booking
3. Book a Bike
4. View My Bookings
5. Cancel a Booking
6. Simple User Registration/Login

### Out of Scope

- Payments
- Admin dashboards
- Bike maintenance tracking
- Advanced authentication
- Mobile applications
- Production deployment

## Acceptance Criteria

- Both client and server are running without errors
- You can explain the role of each folder in the project
- Feature branch created and ready for development
